术语表

术语表

分析/分词 analysis

分词是将全文本转换为词项的过程。根据使用哪种分析器，这些短语：FOO-BAR，Foo-Bar，foo,bar 可能都会被分词为 foo 和 bar。这些词项才是实际上存储在索引中的数据。

针对 FoO:bAR 的全文查询（不是词项查询）也将被分词为词项Foo，bar，从而匹配索引中存储的词项。

正是这种分词过程（在索引时和搜索时都有）使Elasticsearch可以执行全文本查询。

另见文本和词项。

API秘钥 API key

提交Elasticsearch请求时可用于身份验证的唯一标识符。启用TLS时，必须使用基本身份验证（用户名和密码）或API密钥对所有请求进行身份验证。

集群 cluster

共享相同群集名称的一个或多个节点。每个集群都有一个独立的主节点，该节点由集群自动选择，如果失败，可被替换。

组件模板 component template

用于构造索引模板的构造块，用于指定索引映射、设置和别名。

跨群集复制（CCR）

使您能够将远程集群中的索引复制到本地集群的特性。有关更多信息，请参阅跨群集复制。

跨群集搜索（CCS）

跨集群搜索特性允许任何节点充当跨多个集群的联合客户端。请参见跨群集搜索。

文档 document

文档是存储在Elasticsearch中的JSON文档。它就像关系数据库中表中的一行。每个文档都存储在一个索引中，并且有一个类型和一个id。

文档是一个JSON对象（在其他语言中也称为hash/hashmap/associative array），它包含零个或多个字段或键值对。

编制索引的原始JSON文档将存储在_source字段中，在获取或搜索文档时默认返回该字段。

字段 field

文档包含字段或键值对的列表。值可以是简单（标量）值（如字符串、整数、日期），也可以是嵌套结构（如数组或对象）。字段类似于关系数据库中表中的列。

每个字段的映射都有一个字段类型（不要与文档类型混淆），它指示可以存储在该字段中的数据类型，例如integer、string、object。映射还允许您定义（除其他外）如何分词字段的值。

过滤器 filter

过滤器是非评分查询，这意味着它不会对文档评分。它只关心回答一个问题——“这个文档匹配吗？”。答案总是一个简单的，二进制的yes或no。这种查询是在一个过滤器上下文中进行的，因此它被称为过滤器。过滤器是对集合包含或排除的简单检查。在大多数情况下，过滤的目标是减少必须检查的文档数量。

刷新 flush

执行一次Lucene提交将事务日志（translog）中的索引更新写入磁盘。因为Lucene提交是一个相对昂贵的操作，Elasticsearch在translog中记录索引和删除操作，并自动将更改批量刷新到磁盘。如果要从崩溃中恢复，可以从translog中重新执行已确认但尚未提交的操作。在升级之前，可以显式调用Flush API以确保所有更改都提交到磁盘。

跟随者索引 follower index

跨群集复制的目标索引。追随者索引存在于本地集群中，复制自先导索引。

强制合并 force merge

手动触发合并以减少索引的每个分片中的段数，并释放已删除文档使用的空间。不应对正在写入的索引进行强制合并。合并通常是自动执行的，但是您可以在滚动后使用强制合并将旧索引中的分片减少为单个片段。请参见强制合并API。

冻结 freeze

使索引为只读并最小化其内存占用。冻结的索引可以被搜索，而无需像关闭的索引要重新启用，但搜索会受到限制，速度可能会较慢。您可以冻结索引，以减少在准备存档或删除旧索引之前保持旧索引可搜索的开销。请参见冻结API。

冻结索引

降低到低开销状态的索引，仍然支持偶尔的搜索。冻结索引使用内存高效的分片实现和节流搜索来节省资源。搜索冻结的索引比重新打开关闭的索引以启用搜索的开销要低。

id

文档的ID标识。文档的index/id必须是唯一的。如果没有提供ID，那么它将自动生成(另请参见路由）

索引 index

JSON文档的优化集合。每个文档都是字段的集合，即包含数据的键值对。

索引就像关系数据库中的表。它有一个包含类型的映射，其中包含索引中的字段。

+索引是一个逻辑命名空间，它映射到一个或多个主分片，并且可以有零个或多个副本分片。

索引别名 index alias

索引别名是用于引用一个或多个现有索引的辅助名称。

大多数Elasticsearch api都接受索引别名来代替索引名。

请参见添加索引别名。

索引模板 index template

定义要应用于与简单命名模式匹配的新索引的设置和映射，例如logs-*。索引模板还可以将生命周期策略附加到新索引。索引模板用于自动配置滚动期间创建的索引。

先导索引 leader index

跨群集复制的源索引。先导索引存在于远程集群上，并复制给跟随者索引。

本地集群 local cluster

在跨集群搜索或跨集群复制中从远程集群提取数据的集群。

映射 mapping

映射类似于关系数据库中的模式定义。每个索引都有一个映射，该映射定义了一个类型，以及许多索引范围的设置。

映射可以显式定义，也可以在对文档进行索引时自动生成。

节点 node

节点是属于集群的Elasticsearch的运行实例。出于测试目的，可以在一台服务器上启动多个节点，但通常每个服务器应该只有一个节点。

在启动时，节点将使用单播发现具有相同集群名称的现有集群，并尝试加入该集群。

主分片 primary shard

每个文档都存储在一个单独的主分片中。当你索引一个文档时，它首先索引在主分片上，然后再索引到主分片的所有副本上。

默认情况下，索引有一个主分片。您可以指定更多的主分片来缩放索引可以处理的文档数。

一旦创建索引，就不能更改索引中主分片的数量。但是，可以使用split API将索引拆分为新索引。

另请参见路由

查询 query

从Elasticsearch获取信息的请求。您可以将查询视为一个问题，以Elasticsearch理解的方式编写。搜索由一个或多个查询组合而成。

有两种类型的查询：评分查询和过滤器。有关查询类型的详细信息，请参阅查询和筛选上下文。

恢复 recovery

分片恢复是从主分片同步副本分片的过程。完成后，副本分片可用于搜索。

在以下过程中自动进行恢复：

  • 节点启动或失败。这种类型的恢复称为本地存储恢复。
  • 主分片复制。
  • 将分片重新定位到同一集群中的不同节点。
  • 快照恢复。

重新索引 reindex

循环浏览一个或多个索引中的某些或所有文档，将它们重新写入本地或远程集群中的相同或新索引。这通常用于更新映射，或者在两个不兼容的索引版本之间升级Elasticsearch。

远程群集 remote cluster

一个单独的集群，通常位于不同的数据中心或区域，其中包含可由本地集群复制或搜索的索引。到远程群集的连接是单向的。

副本分片

每个主分片可以有零个或多个副本。副本是主分片的副本，有两个用途：

  1. 增加故障转移：如果主分片失败，可以将副本分片提升为主分片

  2. 提高性能：获取和搜索请求可以由主分片或副本分片处理。

默认情况下，每个主分片都有一个副本，但是可以在现有索引上动态更改副本的数量。副本分片永远不会在与其主分片相同的节点上启动。

滚动 rollover

重定向别名以在现有索引达到某个时间段、文档数或大小时开始写入新索引。新索引将根据任何匹配的索引模板自动配置。例如，如果要索引日志数据，可以使用滚动更新创建每日或每周索引。请参见滚动索引API。

汇总 rollup

将高粒度数据汇总为更压缩的格式，以经济高效的方式维护对历史数据的访问。

汇总索引 rollup index

一种特殊类型的索引，用于以较小的粒度存储历史数据。文档由汇总作业汇总并索引到汇总索引中。

汇总作业 rollup job

一种连续运行的后台任务，用于将文档汇总到索引中，并将汇总索引到单独的汇总索引中。作业配置控制汇总哪些信息以及汇总的频率。

路由 routing

当你索引文档时，它存储在单个主分片上。通过哈希routing值来选择该分片。默认情况下，routing值是从文档的ID派生的，如果文档有指定的父文档，则从父文档的ID派生（以确保子文档和父文档存储在同一个分片上）。

可以通过在索引时指定routing值或通过映射中的路由字段来覆盖此值。

分片 shard

分片是单个Lucene实例。它是由Elasticsearch自动管理的低等级“工作者”单元。索引是指向主分片和副本分片的逻辑命名空间。

+除了定义索引应该具有的主分片和副本分片的数量之外，您不需要直接引用碎片。相反，您的代码应该只处理索引。

+Elasticsearch在群集中的所有节点之间分配碎片，并且可以在节点故障或添加新节点的情况下自动将碎片从一个节点移动到另一个节点。

收缩 shrink

减少索引中主分片的数量。当请求数量下降时，可以收缩索引以减少其开销。例如，当索引不再是写入索引时，可以选择收缩索引。请参见收缩索引API。

源字段 source field

默认情况下，索引的JSON文档将存储在_source字段中，并由所有get和search请求返回。这允许您直接从搜索结果访问原始对象，而不需要再一步通过ID检索对象。

词项 term

词项是在Elasticsearch中索引的精确值。词项 foo，Foo，FOO不是等价的。可以使用词项查询来搜索词项（即精确值）。

另见文本和分词。

文本 text

文本（或全文）是普通的非结构化文本，如本段。默认情况下，文本将被分词为词项，这是实际存储在索引中的内容。

文本字段需要在索引时被分词，才能作为全文进行搜索，而全文查询中的关键字必须在搜索时进行分词，以生成（并搜索）索引时生成的相同词项。

另见词项和分词。

类型 type

用于表示文档类型的类型，例如email、user或tweet。类型已弃用且正在删除。请参见移除映射类型。
