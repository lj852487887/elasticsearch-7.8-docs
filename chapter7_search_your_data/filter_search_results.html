筛选搜索结果

Filter search results过滤搜索结果

可以使用两种方法筛选搜索结果：

  • 使用带filter子句的布尔查询。搜索请求对搜索命中和聚合应用布尔过滤器(boolean filter)。
  • 使用搜索API的post_filter参数。搜索请求仅对搜索命中应用后期筛选，而不是聚合。您可以使用后过滤器根据更广泛的结果集计算聚合，然后进一步缩小结果范围。

您还可以在后过滤器之后对命中结果进行rescore，以提高相关性并重新排序结果。

后过滤器

使用post_filter参数筛选搜索结果时，筛选搜索命中发生在计算聚合之后。后过滤器对聚合结果没有影响。

例如，假设您在销售的衬衫具有以下特性：

PUT /shirts
{
  "mappings": {
    "properties": {
      "brand": { "type": "keyword"},
      "color": { "type": "keyword"},
      "model": { "type": "keyword"}
    }
  }
}

PUT /shirts/_doc/1?refresh
{
  "brand": "gucci",
  "color": "red",
  "model": "slim"
}

假设用户指定了两个过滤器：

color:red 和 brand:gucci. 你只想在搜索结果中给他们看Gucci生产的红色衬衫。通常你会使用 bool查询：

GET /shirts/_search
{
  "query": {
    "bool": {
      "filter": [
        { "term": { "color": "red"   }},
        { "term": { "brand": "gucci" }}
      ]
    }
  }
}

但是，您还希望使用分面导航来显示用户可以点击的其他选项的列表。也许您有一个model字段，允许用户将搜索结果限制为 红色Gucci t-shirts 或 dress-shirts。

这可以通过 terms 聚合 来完成：

GET /shirts/_search
{
  "query": {
    "bool": {
      "filter": [
        { "term": { "color": "red"   }},
        { "term": { "brand": "gucci" }}
      ]
    }
  },
  "aggs": {
    "models": {
      "terms": { "field": "model" } ①
    }
  }
}

①返回最流行Gucci的红色衬衫款式。

但也许你也想告诉用户有多少其他颜色的古驰衬衫。如果您只是在color字段上添加一个term聚合，您将只返回red颜色，因为您的查询只返回Gucci的红色衬衫。

相反，您希望在聚合期间包含所有颜色的衬衫，然后仅对搜索结果应用colors过滤器。这就是post_filter的用途：

GET /shirts/_search
{
  "query": {
    "bool": {
      "filter": {
        "term": { "brand": "gucci" } ①
      }
    }
  },
  "aggs": {
    "colors": {
      "terms": { "field": "color" } ②
    },
    "color_red": {
      "filter": {
        "term": { "color": "red" } ③
      },
      "aggs": {
        "models": {
          "terms": { "field": "model" } ③
        }
      }
    }
  },
  "post_filter": { ④
    "term": { "color": "red" }
  }
}

① 主查询现在只查找所有Gucci的衬衫，不管什么颜色

② colors 聚合返回Gucci衬衫的流行颜色

③ color_red 聚合限制了 models 子聚合为 红色 Gucci 衬衫。

④ 最后，post_filter 从搜索 hits 中移除红色外的其他颜色

对筛选搜索结果重新算分

通过对query和post_filter阶段返回的顶部文档（例如100-500）重新排序，这种重算分可以帮助提高精度。重算分是使用二级（通常更昂贵）算法，而不是将昂贵的算法应用于索引中的所有文档，

在每个分片上执行rescore请求，然后返回结果，由处理整个搜索请求的节点对结果进行排序。

目前重算分API只有一个实现: 查询重算分器，它使用一个查询来调整评分。在将来，或许可以提供替代的重算分器，例如，成对重算分器。

如果为rescore查询提供了显式 sort（降序的_score除外），则会抛出错误。

当向用户展示分页时，您不应该在逐步浏览每个页面时更改window_size（通过传递不同的from值），因为这可能会改变顶部的名字结果，从而导致结果在用户逐步浏览页面时发生混乱的移动。

查询重算分

查询重新排序器仅对query和post_filter阶段返回的Top-K结果执行二级查询。每个分片上要检查的文档数可以由window_size参数控制，该参数默认为10。

默认情况下，原始查询和重新存储查询的分数线性组合，以生成每个文档的最终_score。原始查询和重新存储查询的相对重要性可以分别用查询权重和重新存储查询权重来控制。两者都默认为1。

例如：

POST /_search
{
   "query" : {
      "match" : {
         "message" : {
            "operator" : "or",
            "query" : "the quick brown"
         }
      }
   },
   "rescore" : {
      "window_size" : 50,
      "query" : {
         "rescore_query" : {
            "match_phrase" : {
               "message" : {
                  "query" : "the quick brown",
                  "slop" : 2
               }
            }
         },
         "query_weight" : 0.7,
         "rescore_query_weight" : 1.2
      }
   }
}

分数的组合方式可以通过score_mode进行控制：

分数模式 描述
total 添加原始分数和重新算分查询分数。默认值。
multiply 将原始分数乘以重新算分查询分数。用于 function query 重算分。
avg 取原始分数和重新算分查询分数的平均值
max 取原始分数和重新算分查询分数的最大值
min 取原始分数和重新算分查询分数的最小值

多个重新算分

也可以按顺序执行多个重新扫描：

POST /_search
{
   "query" : {
      "match" : {
         "message" : {
            "operator" : "or",
            "query" : "the quick brown"
         }
      }
   },
   "rescore" : [ {
      "window_size" : 100,
      "query" : {
         "rescore_query" : {
            "match_phrase" : {
               "message" : {
                  "query" : "the quick brown",
                  "slop" : 2
               }
            }
         },
         "query_weight" : 0.7,
         "rescore_query_weight" : 1.2
      }
   }, {
      "window_size" : 10,
      "query" : {
         "score_mode": "multiply",
         "rescore_query" : {
            "function_score" : {
               "script_score": {
                  "script": {
                    "source": "Math.log10(doc.count.value + 2)"
                  }
               }
            }
         }
      }
   } ]
}

第一个获取查询结果，然后第二个获取第一个的结果，以此类推。第二个重新排序将“看到”第一个重新排序所做的排序，因此可以使用第一个重新排序上的一个大窗口将文档拉到一个较小的窗口中进行第二个重新排序。
