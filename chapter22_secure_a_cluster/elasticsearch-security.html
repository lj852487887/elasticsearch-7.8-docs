概述

安全概述

Elasticsearch集群通常由许多运动部件组成。有组成集群的Elasticsearch节点，还通常有Logstash实例、Kibana实例、Beats代理和客户端，它们都与集群通信。保护这样的集群有很多方面和层次，这并不奇怪。

安全性通过以下方式保护Elasticsearch集群：

  • 通过密码保护、基于角色的访问控制和IP过滤防止未经授权的访问。
  • 使用SSL/TLS加密保护数据的完整性。
  • 维护审计跟踪，以便您知道谁在对集群及其存储的数据做什么。

防止未经授权的访问

为了防止未经授权访问Elasticsearch集群，您必须有一种对用户进行身份验证的方法。这仅仅意味着您需要一种方法来验证用户是否就是他们声称的用户。例如，您必须确保只有名为Kelsey Andorra的人可以作为用户kandorra登录。Elasticsearch安全功能提供独立的身份验证机制，使您能够快速对集群进行密码保护。如果您已经在使用LDAP、Active Directory或PKI管理组织中的用户，则安全功能可以与这些系统集成以执行用户身份验证。

在许多情况下，仅仅对用户进行身份验证是不够的。您还需要一种方法来控制用户可以访问哪些数据以及可以执行哪些任务。Elasticsearch安全功能允许您通过将访问权限分配给角色并将这些角色分配给用户来授权用户。例如，这种基于角色的访问控制机制（又称RBAC）允许您指定用户kandorra只能对events索引执行读取操作，而不能对其他索引执行任何操作。

请参阅用户身份验证和用户授权。

安全功能还允许您基于IP筛选器限制可以连接到集群的节点和客户端。您可以阻止并允许特定IP地址、子网或DNS域来控制对集群的网络级访问。

保持数据完整性

安全的一个关键部分是对机密数据保密。Elasticsearch具有防止意外数据丢失和损坏的内置保护。然而，没有什么可以阻止蓄意篡改或数据拦截。Elastci Stack安全功能通过加密集群之间、来自集群以及集群内部的通信来保护数据的完整性。请参阅加密通信。为了获得更大的保护，您可以增加加密强度。

保持审计跟踪

保持系统安全需要警惕。通过使用 Elastic Stack 安全特性来维护审计跟踪，您可以很容易地看到谁在访问您的集群以及他们在做什么。您可以配置审核级别，该级别说明记录的事件类型。这些事件包括身份验证尝试失败、用户访问被拒绝、节点连接被拒绝等。通过分析访问模式和访问群集的失败尝试，您可以深入了解尝试的攻击和数据泄露。在集群中保留活动的可审核日志也有助于诊断操作问题。有关更多信息，请参阅审核日志记录。
