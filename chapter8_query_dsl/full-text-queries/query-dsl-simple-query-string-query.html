简单查询字符串

Simple query string query 简单查询字符串查询

使用语法有限但容错的解析器，基于提供的查询字符串返回文档。

此查询使用简单的语法来解析所提供的查询字符串，并根据特殊运算符将其拆分为关键字。然后，查询在返回匹配文档之前独立地分析每个关键字。

虽然它的语法比query_string查询更受限制，但是simple_query_string查询不会因为无效语法而返回错误。相反，它会忽略查询字符串的任何无效部分。

请求示例

GET /_search
{
  "query": {
    "simple_query_string" : {
        "query": "\"fried eggs\" +(eggplant | potato) -frittata",
        "fields": ["title^5", "body"],
        "default_operator": "and"
    }
  }
}

simple_query_string的顶级参数

query

（必需，字符串）要解析并用于搜索的查询字符串。请参见简单查询字符串语法。

fields

（可选，字符串数组）要搜索的字段数组。

此字段接受通配符表达式。您还可以使用插入符号(^)表示法提升与特定字段匹配的相关性分数。有关示例，请参见fields参数中的通配符和字段增强。

默认为index.query.default_field 索引设置，默认值为*。*值提取所有符合关键字查询条件的字段，并过滤元数据字段。如果没有指定prefix，则合并所有提取的字段以生成查询。

一次可以查询的字段数量有限制。它是由indices.query.bool.max_clause_count 搜索设置 决定的，默认为1024。

default_operator

（可选，字符串）如果未指定运算符时，用于解释查询字符串中的文本的默认布尔逻辑。有效值为：

OR (Default)

例如，capital of Hungary的查询字符串被解释为capital OR of OR Hungary。

AND

例如，capital of Hungary的查询字符串被解释为capital AND of AND Hungary。

analyze_wildcard

（可选，布尔值）如果为true，查询将尝试分析查询字符串中的通配符关键字。默认为false。

analyzer

（可选，字符串）用于将查询字符串中的文本转换为标记的分词器。默认值为为default_field映射的索引时分词器。如果没有映射分词器，则使用索引的默认分词器。

auto_generate_synonyms_phrase_query

（可选，布尔值）如果为true，则自动为多关键字同义词创建match_phrase查询。默认为true。有关示例，请参见同义词。

flags

（可选，字符串）简单查询字符串语法的已启用运算符列表。默认为ALL（所有运算符）。有关有效值，请参见限制运算符。

boost

（可选，浮点）用于减少或增加查询的相关性分数的浮点数。默认为1.0。

提升值相对于默认值1.0。介于0和1.0之间的增强值会降低相关性得分。大于1.0的值将增加相关性得分。

fuzzy_max_expansions

（可选，整数）查询为模糊匹配而扩展的最大关键字数。默认为50。

fuzzy_prefix_length

（可选，整数）为模糊匹配保留不变的起始字符数。默认为0。

fuzzy_transpositions

（可选，布尔值）如果为true，则模糊匹配的编辑包括两个相邻字符的换位（ab→ba）。默认为true。

lenient

（可选，布尔值）如果为true，则忽略基于格式的错误，例如为数字字段提供文本值。默认为false。

minimum_should_match

（可选，字符串）要返回的文档必须匹配的最小子句数。有关有效值和更多信息，请参阅minimum_should_match参数。

quote_field_suffix

（可选，字符串）附加到查询字符串中引用文本的后缀。

您可以使用此后缀对精确匹配使用不同的分析方法。请参见混合精确搜索与词干分析。

提示

简单查询字符串语法

simple_query_string 查询支持以下运算符：

  • +表示 AND 操作
  • | 表示 OR 操作
  • - 否定单个标记
  • " 包住部分标记以表示用于搜索的短语
  • * 在关键字尾表示前缀查询
  • ( 和 ) 表示优先
  • ~N 在单词后面表示编辑距离（模糊性）
  • ~N 在短语后面表示slop数量

要按字面意思使用其中一个字符，请在其前面加上反斜杠（\）对其进行转义。

这些运算符的行为可能因default_operator而异。例如：

GET /_search
{
  "query": {
    "simple_query_string": {
      "fields": [ "content" ],
      "query": "foo bar -baz"
    }
  }
}

此搜索用于返回仅包含foo或bar的文档，且这些文档不包含baz。然而，由于default_operator为OR，此搜索实际上返回的是包含foo或bar的文档以及任何不包含baz的文档。要按预期返回文档，请将查询字符串更改为foo bar+-baz。

限制运算符

您可以使用flags参数来限制简单查询字符串语法所支持的运算符。

要仅显式启用特定运算符，请使用 | 分隔符。例如，flags值为 OR | AND | PREFIX 将禁用除OR、AND和PREFIX之外的所有运算符。

GET /_search
{
  "query": {
    "simple_query_string": {
      "query": "foo | bar + baz*",
      "flags": "OR|AND|PREFIX"
    }
  }
}

有效值

可用的标志有：

  • ALL (默认)

    启用所有可选运算符。

  • AND

    启用+ AND运算符。

  • ESCAPE

    启用\作为转义符。

  • FUZZY

    在单词后启用~N运算符，其中N是一个整数，表示允许的匹配编辑距离。见模糊性。

  • NEAR

    在短语后面启用~N运算符，其中N是匹配标记之间允许的最大位置数。与SLOP同义。

  • NONE

    禁用所有运算符。

  • NOT

    启用- NOT运算符。

  • OR

    启用 \| 或运算符。

  • PHRASE

    启用用于搜索短语的 " 引号运算符。

  • PRECEDENCE

    使用 (和 ) 运算符能够控制运算符优先级。

  • PREFIX

    启用* 前缀运算符。

  • SLOP

    在短语后面启用~N运算符，其中N是匹配标记之间允许的最大位置数。NEAR 的近义词。

  • WHITESPACE

    启用空格作为拆分字符。

fields参数中的通配符和每字段增强

字段可以用通配符指定，例如：

GET /_search
{
  "query": {
    "simple_query_string" : {
      "query":    "Will Smith",
      "fields": [ "title", "*_name" ] ①
    }
  }
}

①查询 title, first_name 和 last_name 字段

单个字段可以用插入符号(^)提升相关性：

GET /_search
{
  "query": {
    "simple_query_string" : {
      "query" : "this is a test",
      "fields" : [ "subject^3", "message" ] 
    }
  }
}

① subject 字段 比 message 字段 重要三倍

同义词

simple_query_string查询支持使用 synonym_graph 过滤器进行多词项同义词扩展。使用此过滤器时，解析器为每个多词项同义词创建一个短语查询。例如，以下同义词："ny，new york" 将产生：

(ny OR ("new york"))

也可以将多词项同义词与连词匹配：

GET /_search
{
   "query": {
       "simple_query_string" : {
           "query" : "ny city",
           "auto_generate_synonyms_phrase_query" : false
       }
   }
}

上面的示例创建了一个布尔查询：

(ny OR (new AND york)) city)

匹配有词项ny或连词new AND york的文档。默认情况下，参数auto_generate_synonyms_phrase_query设置为true。
