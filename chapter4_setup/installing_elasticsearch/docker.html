使用Docker安装Elasticsearch

使用 Docker 安装 Elasticsearch

Elasticsearch也可以作为Docker镜像提供。镜像使用centos:7作为基本镜像。

所有已发布的Docker镜像和标记位于www.docker.elastic.co. 源代码在Github中。

这些镜像可以在Elastic许可证下免费使用。它们包含开源和免费的商业功能以及付费商业功能。开始为期30天的试用，试用所有付费商业功能。有关Elastic许可级别的信息，请参阅订阅页。

拉取镜像

获取Docker版本的Elasticsearch只需要对Elastic Docker注册中心发出Docker pull命令。

docker pull docker.elastic.co/elasticsearch/elasticsearch:7.8.1

或者，您可以下载其他Docker镜像，这些映像只包含Apache 2.0许可证下可用的功能。要下载镜像，请转到www.docker.elastic.co.

使用Docker启动一个单节点集群

要启动单节点Elasticsearch集群进行开发或测试，请指定单节点发现以绕过引导检查：

docker run -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" docker.elastic.co/elasticsearch/elasticsearch:7.8.1

使用Docker Compose启动一个多节点集群

要在Docker中建立并运行一个三节点的Elasticsearch集群，可以使用Docker Compose：

创建一个 docker-compose.yml 文件

version: '2.2'
services:
  es01:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.8.1
    container_name: es01
    environment:
      - node.name=es01
      - cluster.name=es-docker-cluster
      - discovery.seed_hosts=es02,es03
      - cluster.initial_master_nodes=es01,es02,es03
      - bootstrap.memory_lock=true
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - data01:/usr/share/elasticsearch/data
    ports:
      - 9200:9200
    networks:
      - elastic
  es02:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.8.1
    container_name: es02
    environment:
      - node.name=es02
      - cluster.name=es-docker-cluster
      - discovery.seed_hosts=es01,es03
      - cluster.initial_master_nodes=es01,es02,es03
      - bootstrap.memory_lock=true
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - data02:/usr/share/elasticsearch/data
    networks:
      - elastic
  es03:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.8.1
    container_name: es03
    environment:
      - node.name=es03
      - cluster.name=es-docker-cluster
      - discovery.seed_hosts=es01,es02
      - cluster.initial_master_nodes=es01,es02,es03
      - bootstrap.memory_lock=true
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - data03:/usr/share/elasticsearch/data
    networks:
      - elastic

volumes:
  data01:
    driver: local
  data02:
    driver: local
  data03:
    driver: local

networks:
  elastic:
    driver: bridge

这个Docker Compose示例文件提供了一个三节点的Elasticsearch集群。节点es01监听localhost:9200，es02和es03通过Docker网络与es01通信。

请注意，此配置在所有网络接口上公开端口9200，并且考虑到Docker如何在Linux上操纵iptables，这意味着您的Elasticsearch集群可以公开访问，可能会忽略任何防火墙设置。如果不想公开端口9200，而是使用反向代理，请将docker-compose.yml文件中的9200:9200替换为127.0.0.1:9200:9200。Elasticsearch将只能从主机本身访问。

被Docker称为为卷(volumes) 的 data01、data02和data03存储着节点数据目录，以便数据在重新启动时保持不变。如果它们还不存在，docker-compose会在您启动集群时创建它们。

1、确保Docker引擎分配了至少4GB的内存。在Docker Desktop中，您可以在“首选项” 的高级选项卡（macOS）或“设置”（Windows）上配置资源使用情况。

Docker Compose未与Linux上的Docker一起预装。有关安装说明，请参见docs.docker.com：在Linux上安装Compose

2、运行 docker-compose 来启动集群：

docker-compose up

3、提交 _cat/nodes 请求以查看节点是否已启动并正在运行：

curl -X GET "localhost:9200/_cat/nodes?v&pretty"

日志消息进入控制台，由配置的Docker日志驱动程序处理。默认情况下，您可以使用 docker logs 命令访问日志。

要停止集群，请运行docker-compose down。使用docker-compose up重新启动集群时，Docker卷中的数据将被保留和加载。要在关闭集群时删除数据卷，请指定 -v 选项：docker-compose down -v。

TLS启用时启动一个多节点集群

请参阅 在Elasticsearch Docker 容器中加密通信 ，以及 在启用TLS的Docker中运行Elastic Stack。

在生产环境使用Docker镜像

以下要求和建议适用于在生产环境中的Docker运行Elasticsearch。

设置 vm.max_map_count 至少为 262144

为了生产环境使用，vm.max_map_count 核心设置必须至少设为262144。

怎么设置 vm.max_map_count 取决于你的平台：

  • Linux

vm.max_map_count 设置应该在 /etc/sysctl.conf 中被永久设置 ：

grep vm.max_map_count /etc/sysctl.conf
vm.max_map_count=262144

如果在一个已启动的系统启用设置，运行：

sysctl -w vm.max_map_count=262144
  • 使用 Doker for Mac 的 macOS

必须在xhyve虚拟机中设置vm.max_map_count设置：

a. 在命令行中运行：

screen ~/Library/Containers/com.docker.docker/Data/vms/0/tty

b. 按下回车键，并使用‘sysctl’ 配置 vm.max_map_count：

sysctl -w vm.max_map_count=262144

c. 离开 screen session , 输入 Ctrl a d .

  • 使用 Doker Desktop 的 Windows 和 macOS

必须通过docker-machine设置vm.max_map_count设置：

docker-machine ssh
sudo sysctl -w vm.max_map_count=262144

配置文件必须能够被 elasticsearch 用户读取

默认情况下，Elasticsearch以用户elasticsearch在容器内运行，使用 uid:gid 1000:0 。

一个例外是Openshift，它使用任意分配的用户ID运行容器。Openshift显示gid设置为0的持久卷，它不做任何调整。

如果要绑定挂载本地目录或文件，elasticsearch用户必须能够读取该目录或文件。此外，此用户必须具有对数据和日志目录的写入权限。一个好的策略是为本地目录授予gid 0的组访问权。

例如，通过绑定装载来准备存储数据的本地目录：

mkdir esdatadir
chmod g+rwx esdatadir
chgrp 0 esdatadir

最后，您可以通过环境变量TAKE_FILE_OWNERSHIP强制容器更改用于数据和日志目录的任何绑定装载的所有权。当你这么做的时候，它们将属于uid:gid 1000:0，它提供对Elasticsearch进程所需的读/写访问权限。

增加nofile和nproc的ulimits

必须为Elasticsearch容器提供nofile和nproc的增加ulimits。验证Docker守护进程的init系统是否将它们设置为可接受的值。

要检查ulimits的Docker守护程序默认值，请运行：

docker run --rm centos:7 /bin/bash -c 'ulimit -Hn && ulimit -Sn && ulimit -Hu && ulimit -Su'

如果需要，可以在守护进程中调整它们，或者在每个容器中重写它们。例如，使用docker run时，设置：

--ulimit nofile=65535:65535

禁用内存交换

为了提高性能和节点稳定性，需要禁用内存交换。有关执行此操作的方法的信息，请参见禁用内存交换。

如果您选择bootstrap.memory_lock:true方法，还需要在Docker守护进程中定义memlock:true ulimit，或者显式地为容器设置memlock:true ulimit，如示例compose文件中所示。使用docker run时，可以指定：

-e "bootstrap.memory\u lock=true" --ulimit memlock=-1:-1

随机化已发布的端口

此镜像暴露了TCP端口9200和9300。对于生产集群，建议使用--publish all随机化已发布的端口，除非您为每个主机固定一个容器。

设置堆大小

要配置堆大小，可以在/usr/share/elasticsearch/config/JVM.options.d下绑定装载一个JVM选项文件，其中包含所需的堆大小设置。请注意，虽然default root jvm.options文件将默认堆设置为1 GB，但在绑定装载的jvm options文件中设置的任何值都将覆盖它。

虽然通过绑定挂载的JVM选项设置堆大小是推荐的方法，但是您也可以通过使用ES\u JAVA\u OPTS环境变量来设置堆大小来进行配置。例如，要使用16 GB，请使用docker run指定-e ES\u JAVA\u OPTS=“-Xms16g-Xmx16g”。请注意，虽然default root jvm.options文件将默认堆设置为1 GB，但您在ES\u JAVA\u OPTS中设置的任何值都将覆盖它。上面的docker-compose.yml文件将堆大小设置为512MB。

即使对容器限制内存访问，也必须配置堆大小。

将部署固定到特定镜像版本

将部署固定到特定版本的Elasticsearch Docker映像。例如docker.elastic.co/elasticsearch/elasticsearch:7.8.1

始终绑定数据卷

出于以下原因，您应该在/usr/share/elasticsearch/data上使用卷绑定：

  1. 如果容器被杀死，Elasticsearch节点的数据不会丢失
  2. Elasticsearch对I/O敏感，Docker存储驱动程序不适合快速I/O
  3. 它允许使用高级Docker卷插件

避免使用loop-lvm模式

如果您使用的是devicemapper存储驱动程序，请不要使用默认的loop-lvm模式。配置docker引擎以使用direct-lvm。

集中日志

考虑使用不同的日志驱动程序来集中日志。还要注意，默认的json文件日志驱动程序并不适合生产使用。

使用Docker配置Elasticsearch

在Docker中运行时，将从/usr/share/Elasticsearch/config/加载Elasticsearch配置文件。

要使用自定义配置文件，请将绑定挂载文件到镜像中的配置文件上。

您可以使用Docker环境变量设置单个Elasticsearch配置参数。示例compose文件和单节点示例使用此方法。

要使用文件内容设置环境变量，请在环境变量名称后面加上 _FILE 。这对于将密码等秘密传递给Elasticsearch而不直接指定它们非常有用。

例如，要从文件设置Elasticsearch引导密码，可以绑定挂载文件并将ELASTIC_PASSWORD_FILE环境变量设置为挂载的位置。如果将密码文件挂载到/run/secrets/password.txt，请指定：

-e ELASTIC_PASSWORD_FILE=/run/secrets/bootstrapPassword.txt

您还可以覆盖镜像的默认命令，将Elasticsearch配置参数作为命令行选项传递。例如：

docker run <various parameters> bin/elasticsearch -Ecluster.name=mynewclustername

虽然绑定挂载配置文件通常是生产环境中的首选方法，但也可以创建包含配置的自定义Docker镜像。

挂载Elasticsearch配置文件

创建自定义配置文件并将其绑定到Docker映像中的相应文件上。例如，要将mount custom\u elasticsearch.yml与docker run绑定，请指定：

-v full_path_to/custom_elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml

容器以用户elasticsearch的身份使用uid:gid 1000:0运行Elasticsearch, 绑定挂载的主机目录和文件必须可由该用户访问，并且数据和日志目录必须可由该用户写入。

挂载Elasticsearch秘钥存储

默认情况下，Elasticsearch将自动生成密钥存储文件以进行安全设置。此文件混淆，但未加密。如果要使用密码加密安全设置，必须使用elasticsearch-keystore功能创建受密码保护的密钥存储库，并将其绑定挂载到容器中，设置为/usr/share/elasticsearch/config/elasticsearch.keystore。为了在启动时向Docker容器提供密码，请将Docker环境值KEYSTORE_PASSWORD设置为您的密码的值。例如，docker run命令可能具有以下选项：

-v full_path_to/elasticsearch.keystore:/usr/share/elasticsearch/config/elasticsearch.keystore
-E KEYSTORE_PASSWORD=mypassword

使用自定义Docker镜像

在某些环境中，准备包含配置的自定义映像可能更适用。实现此目的的Dockerfile可能会简单如下：

FROM docker.elastic.co/elasticsearch/elasticsearch:7.8.1
COPY --chown=elasticsearch:elasticsearch elasticsearch.yml /usr/share/elasticsearch/config/

然后这样可以构建与运行镜像：

docker build --tag=elasticsearch-custom .
docker run -ti -v /usr/share/elasticsearch/data elasticsearch-custom

一些插件需要额外的安全权限。您必须通过以下方式明确接受它们：

  • 运行Docker镜像时附加tty并在提示时允许权限。
  • 通过向plugin install命令添加--batch标志来检查安全权限并接受它们（如果合适）。

有关更多信息，请参阅插件管理。
